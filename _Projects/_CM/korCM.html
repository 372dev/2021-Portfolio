  1. 프로젝트 개요

  2. KH 정보교육원 응용 SW 개발자 과정중 진행한 팀 프로젝트입니다.
  3. 공공 API(공연예술통합전산망/KOPIS)를 활용하여 뮤지컬, 연극, 클래식을 아우르는 전국 공연 정보를 모아 사용자가 쉽게 조회 및 검색할 수 있고, 간편하게 예매도 할 수 있는 서비스를 목표로 구현하였습니다.
  4. 실제 공연 정보 조회(목록조회, 상세조회, 공연장 정보조회) 등을 제공하고 선택된 공연에 대한 좌석선택, 예매, 결제 서비스를 제공합니다. 또한 공연 박스오피스, 공연 리뷰, 평점 작성, 게시판을 활용한 커뮤니티 기능, 고객센터, 관리자 페이지, 마이페이지(회원정보 관리, 자기가 쓴 글/리뷰 관리) 등의 다양한 기능이 유연하게 작동하도록 구현하였습니다.
  5. 저는 팀장의 직무를 맡았으며 담당한 핵심 기능은 공공 API 활용 입니다. 담당 업무 상세는 아래와 같습니다.
  6. 기획단계 : 주제선정, 기획의도/PPT초안, 기능정의, 화면 설계, DB 설계, 회의 진행
  7. 구현단계 : 메인화면, 공연 목록(공공API), 공연 검색(공공API & AJAX), 박스오피스(공공API & 크롤링), 잘못된 접근 페이지, 메이트-공연정보 표시(공공API), 예매확인증 QR 코드 이메일 발송(ZXing & MailHandler)
  8. 교육과정에서 배운 기술을 활용하며 새로운 기술도 다양하게 시도해 보는 것에 프로젝트의 중점을 두었습니다. 나아가 깃허브의 브랜치, PR, 코드리뷰 등의 기능을 적극적으로 활용하여 버전관리와 협업을 성공적으로 마쳤습니다.

Page 1

  1. 서비스 랜딩 페이지

  2. 페이지 상단에는 트리구조의 사이드 메뉴가 포함된 슬라이더가 위치하고, 슬라이더에는 프로모션 내용이 담겨있으며 클릭시 프로모션 페이지로 이동합니다
  3. 그 아래에는 검색창이 있는데 공연 제목으로 검색을 할 경우 공연 목록/검색 페이지로 이동하여 곧바로 해당 검색 결과를 표시합니다. 이 때, 장르 선택도 가능합니다(전체/뮤지컬/연극/클래식).
  4. 그 아래에는 뮤지컬/연극/클래식/메이트게시판 등의 서비스로 빠르게 이동할 수 있는 아이콘 링크가 있습니다.
  5. 그 아래에는 후술될 Rest API 크롤링으로 저장된 뮤지컬/연극/클래식 세가지 공연 장르의 박스오피스 정보중 상위 3개의 공연 순위가 장르별로 각각 표시되며 1~3위까지의 순위 번호가 포스터 위에 이미지로 표시됩니다.
  6. 더 보기 버튼을 누르면 해당 박스오피스 페이지로 이동하고 공연 포스터를 누르면 해당 공연 상세조회 페이지로 이동합니다.

  7. 활용한 기술

  8. Spring, jQuery, Bootstrap, AJAX, Rest API

Page 2

  1. 박스오피스 페이지

  2. 컨트롤러의 스케쥴러에 의해 지정된 시간에 메서드가 실행되면 RestTemplate을 활용하여 공공 API에서 공연 예매 순위정보를 가져와서 DB에 저장합니다.
  3. 헤더 메뉴의 링크, 메인페이지의 박스오피스 더보기 링크 등을 통해 박스오피스 조회 요청을 받는 경우 DB에 크롤링된 정보를 myBatis를 통해 가져온 뒤 객체 리스트에 담아 EL-JSTL 구문으로 화면에 표시합니다.
  4. 뮤지컬, 연극, 클래식 각각의 박스오피스 페이지를 요청 파라미터에 따라 하나의 view.jsp로 부터 JSTL 조건문을 이용하여 생성합니다.
  5. 검색된 공연목록의 포스터를 클릭하면 해당 공연 상세조회 페이지로 이동하며 하단에 더 보기 버튼을 클릭하면 공연 검색 페이지로 이동합니다.

  6. 활용한 기술

  7. Spring, jQuery, Bootstrap, AJAX, Rest API, myBatis, Oracle DB

Page 3

  1. 공연 목록 조회/검색 페이지

  2. RestTemplate을 활용하여 공공API에서 공연 목록 정보를 가져와 객체 리스트에 담아 EL-JSTL 구문으로 화면에 표시합니다.
  3. 이후 검색을 하거나 장르 선택등의 추가 요청은 AJAX를 통해 컨트롤러로 요청을 전달하여 새로운 정보를 화면에 업데이트 합니다.
  4. 상세검색 조건은 버튼 토글을 활용하여 숨기거나 펼칠 수 있으며 항목은 공연예정/공연중, 장르(뮤지컬/연극/클래식), 공연명 검색이 있습니다.
  5. 검색된 공연목록의 포스터를 클릭하면 해당 공연 상세조회 페이지로 이동합니다.

  6. 활용한 기술

  7. Spring, jQuery, Bootstrap, AJAX, Rest API

Page 4

  1. 같이가요 게시판 (부분 구현)

  2. 특정한 공연을 함께 관람할 사람을 찾을 수 있는 게시판 서비스입니다. 해당 게시판 관련 기능 중 글쓰기에서는 해당 공연 아이디를 DB에 저장, 글 조회시에는 저장된 공연 아이디를 이용하여 공공 API에서 공연 상세정보를 조회해 오는 기능을 부분 구현 했습니다. 그 외 게시글, 댓글 관련 기능은 프로젝트 팀의 담당 조원이 구현했습니다.
  3. 공연 상세조회 페이지에서 버튼을 클릭하여 같이가요 게시판의 글 작성 페이지로 이동이 가능합니다. 이때, 요청 파라미터로 공연 상세조회의 Rest API 결과중 공연 아이디를 전달하여 게시판 글 작성 form 안에 포함시킵니다.
  4. 이후 작성된 게시글을 조회할 경우 DB에 저장된 공연아이디를 전달하는 AJAX 요청을 이용하여 공연 상세조회 API를 호출하고 게시글 내부 카드 형식으로 결과를 출력합니다.
  5. 게시글 작성시 선택된 공연을 변경하고 싶은 경우 버튼을 이용하여 다시 공연목록 페이지로 이동이 가능합니다.
  6. 게시글 안에 출력된 공연 정보 중 포스터를 클릭하면 해당 공연 상세조회 페이지로 이동합니다.

  7. 활용한 기술

  8. Spring, jQuery, Bootstrap, AJAX, Rest API, JDBC, Oracle DB

Page 5

  1. 잘못된 접근 페이지

  2. HTTP 에러가 발생했을 경우, web.xml 설정에 따라 지정된 컨트롤러로 에러 메세지가 전달됩니다.
  3. 컨트롤러에서 에러 종류에 따라 카테고리로 분류된 메세지를 에러 페이지 view로 전달합니다.
  4. 전달된 메세지 카테고리에 따라 하나의 view에서 JSTL 조건문을 통해 다양한 에러 페이지를 생성하여 화면에 표시합니다.
  5. 카테고리는 4가지로 나뉩니다 - 401&403, 404, 500, 기타 에러

  6. 활용한 기술

  7. Spring

Page 6

  1. 예매확인서 발송

  2. 공연 목록조회, 공연 상세조회를 통해 불러온 공연을 특정 날짜, 인원수를 선택하여 예매할 수 있는 기능은 프로젝트 팀의 담당 조원이 구현했습니다. 저는 여기서 예매와 결제에 에 성공한 경우 사용자에게 예매확인증을 발송하는 기능을 구현했습니다.
  3. 예매에 성공한 경우 예매정보를 담은 QR 코드 이미지를 생성하여 저장합니다(ZXing 라이브러리).
  4. 예매정보 텍스트와 함께 QR 이미지를 첨부하여 사용자에게 이메일을 발송합니다(MailHandler 라이브러리).

  5. 활용한 기술

  6. Spring
